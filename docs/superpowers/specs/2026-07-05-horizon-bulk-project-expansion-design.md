# Horizon 项目批量展开设计

## 目标

在主页面增加一个紧凑的动态按钮，让用户一次展开或收起当前显示的全部周任务与长期任务项目卡片。

## 交互规则

- 按钮只出现在当前任务主页面，不出现在历史记录详情中。
- 当前显示的项目中只要存在一个收起项，按钮文案为“全部展开”；点击后展开全部当前显示项目。
- 当前显示项目全部展开时，按钮文案为“全部收起”；点击后收起全部当前显示项目。
- 搜索或归档筛选生效时，批量操作只影响当前筛选结果中显示的项目，不改变隐藏项目的展开状态。
- 没有任何可折叠项目时按钮保留但禁用，避免页面标题位置跳动。
- 周任务与长期任务使用同一个按钮统一处理；单独点击项目卡片后，批量按钮文案立即重新计算。

## 视觉位置

- 按钮位于主页面“本周任务”标题行右侧。
- 使用现有 `MiniButtonStyle`，保持 Orbit Blue 的圆润扁平视觉和窄面板中的紧凑尺寸。
- 不新增颜色、图标、弹窗或确认步骤。

## 状态与持久化

- 继续复用 `ProjectExpansionRules.SetExpanded` 和 `HorizonSettings.ProjectExpansionStates`。
- 一次批量操作只保存一次数据、只刷新一次页面。
- 周任务和长期任务仍使用独立持久化键；批量操作只是同时更新当前可见键，不改变现有数据格式。
- 保存失败时保留本次内存状态，并显示现有风格的状态提示。

## 验收标准

1. 默认项目均收起时显示“全部展开”。
2. 点击后，当前显示的周任务和长期任务项目全部展开，按钮切换为“全部收起”。
3. 再次点击后全部收起，按钮恢复为“全部展开”。
4. 混合展开状态下显示“全部展开”，点击后统一展开。
5. 单独展开或收起项目后，按钮文案立即更新。
6. 搜索结果中的批量操作不改变隐藏项目状态。
7. 批量状态在重新启动后保持。
8. 历史记录不可折叠行为不受影响。
9. 自动测试、Debug 构建和 Release 发布通过。

## 非目标

- 不增加两个独立按钮。
- 不改变项目卡片自身的点击展开方式。
- 不对历史记录项目执行批量折叠。
