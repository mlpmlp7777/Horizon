# Horizon 新增内容双列菜单设计

## 目标

优化主页面左上角“新增内容”按钮弹出的“新建本周任务 / 新建长期任务”选项，使其与现有 Orbit Blue 扁平、圆润的主页面风格一致，并提升两类任务入口的辨识度。

## 范围

- 只调整“新增内容”弹出菜单及其中两个菜单项。
- 不修改主页面空状态中的“新增本周任务 / 新增长期任务”按钮。
- 不修改任务编辑器、数据模型、保存流程或项目折叠功能。

## 视觉方案

采用已确认的 B 方案“横向双列选择块”：

- 菜单从“新增内容”按钮下方弹出，宽度约 `236px`。
- 外层使用白色背景、Orbit Blue 边框体系、`18px` 圆角和 `8px` 内边距。
- 两个菜单项横向排列、等宽显示，每项约 `104px × 76px`，圆角 `13px`。
- 菜单项默认使用 `OrbitTaskBrush` 浅蓝背景和透明边框。
- 悬停时使用 `OrbitHeaderBrush` 背景及 `OrbitExpandedBorderBrush` 边框。
- 按下时使用 `OrbitPrimaryBrush` 背景和白色文字，不增加渐变或阴影。
- “新建本周任务”使用圆角图标块与“周”字图标。
- “新建长期任务”使用圆角图标块与“∞”图标。
- 保留完整中文文案，不缩写成“本周任务 / 长期任务”。

## 交互规则

- 点击“新增内容”后，菜单固定在该按钮正下方并保留小间距。
- 点击任一菜单项后关闭菜单，并打开对应的新建任务编辑器。
- 点击菜单外部或按 `Esc` 关闭菜单。
- 保留 WPF `ContextMenu` 与 `MenuItem` 的键盘导航、焦点和自动化语义。
- 图标只用于区分类型，不替代文字；自动化名称仍使用完整菜单文案。
- 主面板失去焦点时，如果菜单仍打开，继续遵循现有逻辑，不自动收起面板。

## 实现方式

- 在 `src/Horizon.App/MainWindow.xaml` 中新增有键名的 `QuickAddContextMenuStyle` 和 `QuickAddMenuItemStyle`。
- `ContextMenu` 模板使用圆角 `Border` 与横向 `ItemsPanel`。
- `MenuItem` 模板绘制图标块和完整标题，并包含悬停、按下与禁用状态。
- 在 `src/Horizon.App/MainWindow.xaml.cs` 中继续动态创建菜单，但为菜单和菜单项应用上述资源；菜单项继续复用现有打开编辑器回调。
- 不为整个窗口设置隐式 `MenuItem` 样式，避免影响未来其他菜单。

## 验收标准

1. 菜单在“新增内容”按钮下方打开，呈横向双列卡片布局。
2. 两项完整显示“新建本周任务”和“新建长期任务”。
3. 图标、背景、边框、圆角和状态颜色与 Orbit Blue 主页面一致。
4. 悬停、按下、禁用状态清晰，无渐变、阴影或系统默认灰色菜单外观残留。
5. 两个入口分别打开正确的任务编辑器，点击后菜单关闭。
6. 点击外部和按 `Esc` 可以关闭菜单。
7. 键盘可聚焦并选择两个菜单项，自动化名称保持完整文案。
8. 主页面空状态按钮及其他按钮样式不变。
9. 自动测试、Debug 构建和 Release 发布通过。

## 非目标

- 不增加第三个新增入口。
- 不为菜单项增加说明文字或二级菜单。
- 不重构主页面顶部其他按钮。
