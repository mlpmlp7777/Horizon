# Horizon 自动周转与分层历史设计

## 目标

让本周任务符合每周节奏：周一自动整理上周任务；让已完成的长期任务在下一个周一进入独立长期历史；历史先按周缩略展示，再进入周详情。

## 完成时间

- `WeeklyTaskItem` 和 `LongTermTaskItem` 增加可空的 UTC 字段 `CompletedAt`。
- 状态首次切换为“已完成”时写入当前 UTC 时间。
- 从“已完成”切换回其他状态时清空 `CompletedAt`。
- 旧数据中状态已完成但缺少 `CompletedAt` 时，以 `UpdatedAt` 作为完成时间；迁移后立即保存。

## 周一自动整理

整理以系统本地时间和周一为一周起点：

- 每次程序启动时执行一次。
- 程序持续运行时，每分钟检查本地日期；日期变化后执行一次，因此跨到周一无需重启。
- 如果程序在周一没有运行，下次启动时补做所有必要整理。

整理规则：

- `WeekStartDate` 早于本周且未完成的本周任务，将 `WeekStartDate` 更新为本周周一，继续显示在主页面。
- 已完成的本周任务不顺延；当其 `CompletedAt` 早于本周周一时，从主页面排除并进入每周任务历史。
- 已完成的长期任务在完成当周仍显示在主页面；当其 `CompletedAt` 早于本周周一时，从主页面排除并进入长期任务历史。
- 自动历史不修改现有 `Archived` 字段；手动归档仍是独立的隐藏/恢复机制，当前归档筛选规则继续适用。

## 历史分类与分组

- 历史入口顶部使用两个标签：“每周任务”和“长期任务”。
- 两类任务拥有独立的周卡列表，不混合显示。
- 每周任务历史按任务的 `WeekStartDate` 分组。
- 长期任务历史按 `CompletedAt` 转为本地时间后的周一日期分组。
- 周卡显示周日期范围、任务数、项目数和完成概览。
- 周卡按周开始日期倒序排列。

## 周详情

- 点击周卡进入该类型、该周的详情页。
- 详情顶部显示返回按钮、周日期范围和汇总。
- 任务继续按项目分组，使用与主页面一致的紧凑任务卡。
- 历史详情保留批注入口，可查看和维护任务批注。
- 返回后恢复原历史类型标签和周列表滚动上下文。

## 状态与空态

- 历史首页、周详情、每周标签和长期标签使用明确的 ViewModel 状态，不复用多个互相冲突的布尔值。
- 没有历史时分别显示“暂无每周任务历史”和“暂无长期任务历史”。
- 被恢复为未完成状态的任务立即离开历史；本周任务随后按当前周规则回到主页面，长期任务回到长期任务区。

## 验证

- 上周未完成的本周任务在周一顺延到当前周。
- 上周已完成的本周任务进入每周历史，不再出现在主页面。
- 上周已完成的长期任务进入长期历史，并与每周历史分离。
- 本周刚完成的两类任务在下一个周一之前仍显示在主页面。
- 错过周一运行后，下次启动能正确补做整理。
- 旧完成任务能生成 `CompletedAt` 并进入正确周。
- 历史首页先显示周卡，点击后只显示所选周详情，返回状态正确。
- 批注在主页面和历史详情保持一致。
- 自动测试使用可注入时间覆盖周日到周一、跨多周和状态恢复场景。
- Debug 构建、Release 发布和运行检查通过。

## 非目标

- 不提供自定义周起始日。
- 不创建历史任务副本或不可变快照。
- 不混合展示每周任务历史和长期任务历史。
- 不增加统计报表、图表或导出功能。
