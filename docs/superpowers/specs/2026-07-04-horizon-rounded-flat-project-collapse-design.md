# Horizon 圆润扁平化与项目折叠设计

## 目标

在不改变 Horizon 右侧面板、周任务在上和长期任务在下的基本结构下，完成两项优化：

1. 周任务和长期任务均支持按项目展开或收起；收起后隐藏全部子任务，仅显示项目摘要。
2. 完整面板采用已确认的「环轨蓝」视觉方向：扁平、圆润、有明确产品辨识度，同时保持桌面效率工具所需的信息密度。

## 范围

本次包含完整面板的项目折叠交互、展开状态持久化、颜色与圆角体系、项目卡片和子任务卡片的视觉调整。

本次不修改极细触发条、悬停把手、面板宽度、自动隐藏逻辑、任务数据结构、任务编辑流程、归档流程、历史页面或应用图标。

## 项目折叠交互

### 折叠单位

- 周任务区和长期任务区分别按项目折叠。
- 同一个项目在两个分区中的状态相互独立。例如，周任务中的项目可以收起，而长期任务中的同名项目仍保持展开。
- 点击项目摘要行切换展开和收起状态。
- 项目内部的编辑、归档、状态切换等控件只执行自身操作，不触发项目折叠。

### 收起状态

收起后隐藏该项目下的全部子任务、备注、操作按钮和任务级进度控件。项目摘要仍显示：

- 项目名称
- 项目状态
- 子任务总数与已完成数量
- 汇总进度百分比
- 汇总进度条
- 指向右侧的展开图标

收起状态必须仍能快速判断项目规模与进展，不能退化为只有标题的空白行。

### 展开状态

展开后保留相同的项目摘要，并在下方显示现有子任务内容。展开图标改为向下。任务卡片沿用当前功能，仅调整为新的圆润扁平样式。

### 状态记忆

- 每个项目在周任务区和长期任务区各自拥有一个展开状态。
- 状态切换后立即持久化，应用重启后恢复上次状态。
- 尚无保存记录的项目默认收起，以符合“先看项目，需要时再看子任务”的使用目标。
- 项目被删除或归档后，遗留的展开状态可以在后续保存时清理；无效记录不得影响加载。

## 视觉方向：环轨蓝

### 视觉原则

- 使用纯色块、描边和层级留白，不使用渐变或厚重投影。
- 圆润但不幼态。不同层级使用不同圆角，避免所有元素都成为相同胶囊。
- 主色只用于主要操作、进度和当前状态，避免整面板泛蓝。
- 通过顶部环轨图形、钴蓝强调和瓷白内容区形成稳定辨识度。

### 色彩

- 面板内容底色：`#F7F9FD`
- 顶部品牌区：`#DCE9FF`
- 主强调色：`#2D68FF`
- 主文字：`#14213D`
- 次文字：`#526686`。该值用于 `9–10.5 px` 次级文字，在顶部、面板与任务卡浅色背景上均保持至少 `4.5:1` 对比度。
- 收起项目底色：`#EAF0F9`
- 展开项目底色：`#FFFFFF`
- 展开项目描边：`#9DB9FF`
- 子任务底色：`#EEF3FF`
- 进度轨道：`#D4DEED`
- 装饰点缀色：`#D6FF58`，仅用于顶部环轨图形的小面积信号点。

颜色在 WPF 实现时应集中为资源，禁止在同类控件中散落相近但不同的蓝色。

### 圆角层级

- 完整面板外壳：`30–32 px`
- 项目卡片：`18 px`
- 子任务卡片：`14 px`
- 顶部工具按钮：`12 px`
- 展开图标底板：圆形
- 状态标签和置顶标签：胶囊形

圆角值可以按 WPF 像素对齐微调，但必须保持“外层最大、内容层递减、状态标签为胶囊”的层级关系。

### 顶部与导航

- 保留 Horizon 名称、当周摘要、置顶状态和“新增 / 设置 / 归档 / 历史”四个入口。
- 顶部使用浅蓝纯色品牌区，右上角放置被裁切的环轨圆形作为低对比装饰；装饰不得遮挡文字和点击区域。
- “新增”使用钴蓝底白字，其余入口使用半透明白色或等效浅色纯色块。
- 继续使用中文界面文案；`Horizon` 作为产品名保留。

## 布局

- 完整面板继续保持现有 `360 px` 宽度。
- 周任务和长期任务保持上下分区，不改为标签页。
- 分区标题右侧显示项目数与任务数摘要。
- 项目卡片之间使用 `8 px` 垂直间距。
- 收起项目保持紧凑；展开项目可随子任务数量增长，并由现有滚动容器承载。
- 项目摘要行必须在 `360 px` 宽度下完整容纳展开图标、主信息和状态标签，不允许文字与状态重叠。

## 组件边界

### 项目摘要组件

负责显示项目名称、汇总信息、状态、进度条和展开图标，并发出展开状态切换请求。它不处理任务编辑或归档。

### 子任务列表组件

只在项目展开时渲染，复用当前任务命令和编辑入口。视觉样式随环轨蓝体系调整，但不改变命令语义。

### 展开状态存储

负责以“分区类型 + 项目 ID”为稳定键读取和保存布尔状态。视图模型消费该状态，不直接操作文件。

## 数据流

1. 数据加载后，根据项目 ID 和分区类型恢复展开状态。
2. 项目摘要将恢复结果呈现为展开或收起。
3. 用户点击摘要行后，视图模型先更新当前项目状态，再请求保存设置。
4. 视图根据状态显示或隐藏子任务列表，并切换箭头方向。
5. 任务状态或进度发生变化时，项目摘要立即重新计算任务数、已完成数量与汇总进度，不依赖展开状态。

## 异常与兼容

- 旧版数据没有展开状态时正常加载，并将所有项目视为默认收起。
- 展开状态记录中的未知项目 ID 被忽略，不导致数据加载失败。
- 保存失败时保留当前会话中的视觉状态，并沿用现有错误反馈方式；不得因保存失败阻止用户继续查看任务。
- 项目名称变化不影响状态，因为持久化键使用项目 ID 而不是名称。
- 空项目仍显示 `0 项任务 · 已完成 0 项 · 0%`，并允许展开，但展开区域不生成空白任务卡。

## 验证

- 周任务和长期任务中的任意项目均可独立展开和收起。
- 同一项目在两个分区中的状态不会相互覆盖。
- 重启应用后恢复每个项目上次的展开状态。
- 无历史状态的新项目默认收起。
- 收起后只显示项目摘要，不渲染或占位子任务内容。
- 摘要准确显示项目名、状态、任务总数、完成数量、百分比和进度条。
- 点击项目内部操作按钮不会误触折叠。
- 空项目、已归档项目和失效状态记录不会导致异常。
- `360 px` 宽度下标题、摘要、状态标签和展开图标无重叠或裁切。
- 面板、项目、任务和标签的圆角层级符合本设计；界面无渐变、无厚重阴影。
- 可见界面文案保持中文，产品名 `Horizon` 除外。
- 自动化测试、Debug 构建和 Release 发布通过，并手工验证状态持久化与完整交互闭环。

## 非目标

- 不增加全局“一键全部展开 / 全部收起”。
- 不增加分区整体折叠。
- 不改变任务的三状态模型。
- 不重新设计任务编辑器、历史页面或系统托盘。
- 不将本次 HTML 效果图作为运行时资源打包进 WPF 应用。
