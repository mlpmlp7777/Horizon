# Horizon 顶部布局与置顶设计

## 目标

移除顶部搜索框，增加可持久化的置顶开关，并修复圆角面板左上内容被裁切的问题。

## 布局

- 顶部使用双层工具栏。
- 第一行左侧显示 `Horizon` 和任务摘要，右侧显示“置顶”开关。
- 第二行依次显示“新增内容”“设置”“归档”“历史”。
- 删除搜索框，不在其他位置补充搜索入口。
- 完整面板内部边距调整为顶部 `22 px`、左侧 `18 px`、右侧和底部 `16 px`，避免标题进入圆角裁切区。
- 第二行按钮使用紧凑尺寸，中文标签保持完整可见。

## 置顶行为

- `HorizonSettings` 增加 `IsPinned`，每次切换后立即保存。
- 置顶开启时按钮使用蓝色高亮背景，并显示选中状态。
- 置顶开启时，点击桌面、任务栏或其他应用不收起完整面板。
- 置顶关闭时，继续使用现有的窗口失活自动收起逻辑。
- 无论是否置顶，程序启动时都先显示右侧极细触发条；置顶只影响完整面板打开后的行为。
- 当前窗口继续保持现有 `Topmost` 设置；“置顶”在本版本中表示固定展开，不改变 Windows 窗口层级策略。

## 状态规则

- 窗口失活收起判断增加 `IsPinned` 条件。
- 打开应用内菜单时仍不误收起。
- 取消置顶后，如果窗口随后失活，则正常收起；取消置顶本身不立即关闭面板。

## 验证

- 顶部不显示搜索框。
- 左上标题和摘要完整可见，不被圆角或边框裁切。
- 两行工具栏在 `360 px` 面板宽度下不截断、不重叠。
- 开启置顶后点击外部不收起，关闭后点击外部会收起。
- 置顶状态重启后保留，但启动形态仍为极细触发条。
- 应用内菜单不触发误收起。
- 自动测试、Debug 构建和 Release 发布通过。

## 非目标

- 不恢复或重新设计搜索功能。
- 不改变极细触发条、悬停把手或完整面板宽度。
- 不增加 Windows 托盘功能。
