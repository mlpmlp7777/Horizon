# Horizon 环轨蓝滚动条设计

## 目标

将 Horizon 中仍使用系统默认外观的滚动条统一为“环轨蓝”视觉语言，使其与现有扁平、圆润的主界面协调，同时保持滚动区域轻巧、不抢占任务内容的注意力。

## 适用范围

- 主页面周任务与长期任务区域。
- 历史记录页面。
- 设置及项目管理区域。
- 窗口内由 `ScrollViewer`、`ListBox` 等控件产生的垂直和水平滚动条。

本次只调整滚动条视觉模板与交互状态，不改变滚动逻辑、数据模型、页面结构或业务行为。

## 视觉方案

采用已确认的 A 方案“轻量胶囊”：

- 轨道默认透明，避免在窄侧边栏中形成持续的视觉分隔线。
- 垂直滚动条布局宽度为 `12px`，默认可见滑块宽度为 `6px`，并在轨道内居中。
- 水平滚动条布局高度为 `12px`，默认可见滑块高度为 `6px`，并在轨道内居中。
- 滑块使用现有 `OrbitPrimaryBrush`（`#2D68FF`），两端为完整胶囊圆角。
- 悬停时滑块扩展至 `8px`，轨道显示轻微浅蓝背景，帮助用户识别可操作区域。
- 拖动时滑块使用现有 `OrbitAccentInkBrush`（`#2458D4`），提供清晰但克制的按压反馈。
- 不显示上下或左右箭头按钮。
- 滑块最小长度为 `30px`，避免长内容下滑块过短而难以操作。

## 交互行为

- 鼠标滚轮、触控板、键盘滚动以及拖动滑块的行为保持 WPF 原生语义。
- 点击透明轨道仍执行分页滚动。
- 视觉滑块虽然较窄，但整个 `12px` 轨道区域均可参与鼠标交互。
- 只有当前滚动条进入悬停或拖动状态时才增强显示，不影响同页面其他滚动区域。
- 内容无需滚动时，继续遵循现有 `VerticalScrollBarVisibility="Auto"` 等设置，不占用额外空间。

## 实现方式

- 在 `src/Horizon.App/MainWindow.xaml` 的窗口资源中增加统一的 `Thumb` 与 `ScrollBar` 控件模板。
- 模板复用现有 Orbit Blue 画刷，不引入新的渐变、阴影或独立配色体系。
- 分别处理垂直与水平朝向，保留 WPF `PART_Track`、`DecreaseRepeatButton`、`IncreaseRepeatButton` 和拖动滑块所需绑定。
- 通过隐式 `ScrollBar` 样式覆盖窗口内所有滚动区域，避免逐个页面重复指定样式。
- 不修改 `AGENTS.md`，不改动 ViewModel、持久化或任务折叠功能。

## 验收标准

1. 主页面、历史页和设置区域的滚动条均使用相同的胶囊外观。
2. 默认状态为透明轨道加 `6px` 蓝色滑块。
3. 悬停时滑块扩展至 `8px`，轨道出现浅蓝提示；移出后恢复默认状态。
4. 拖动时滑块变为深蓝，松开后恢复对应状态。
5. 垂直和水平滚动条均无箭头按钮，最小滑块长度不小于 `30px`。
6. 鼠标滚轮、轨道点击、滑块拖动、触控板和键盘滚动正常。
7. 无需滚动时不显示滚动条，不产生额外空白或遮挡任务内容。
8. 项目可折叠、展开状态持久化及历史页只读行为不受影响。
9. `dotnet run --project tests/Horizon.App.Tests/Horizon.App.Tests.csproj` 与 `dotnet build Horizon.sln` 通过。

## 非目标

- 不增加滚动动画、惯性滚动或自动隐藏计时逻辑。
- 不修改页面宽度、卡片间距或任务列表结构。
- 不为单个页面设计不同颜色或尺寸的滚动条。
