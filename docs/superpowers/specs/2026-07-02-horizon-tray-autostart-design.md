# Horizon 托盘驻留与开机启动设计

## 目标

让 Horizon 作为轻量桌面工具常驻 Windows 通知区域，不在任务栏显示；用户可以从托盘快速展开面板，并在设置中控制是否随 Windows 自动启动。

## 窗口与托盘行为

- 主窗口始终设置为不在任务栏显示，现有右侧三态交互保持不变。
- 程序运行期间始终显示一个 Horizon 托盘图标。
- 左键单击托盘图标后，主窗口切换到完整面板状态、显示到当前工作区右侧并获得焦点。
- 右键托盘图标显示“打开 Horizon”和“退出 Horizon”。
- “打开 Horizon”与左键单击行为一致。
- “退出 Horizon”是用户主动结束常驻进程的明确入口；退出前释放托盘图标。
- 点击面板外部仍按置顶状态决定是否收起，不结束程序。
- Windows 会话结束或应用异常退出时，尽力释放托盘资源，不阻碍系统关机。

## 开机自动启动

- 设置页新增“开机自动启动”开关和简短说明。
- 新数据文件中的默认值为开启；旧数据缺少该字段时也按开启迁移。
- 开关变化后立即保存到 Horizon 本地 JSON，并同步当前用户启动项。
- 启动项使用 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`，值名固定为 `Horizon`，命令为带引号的当前可执行文件绝对路径。
- 开机启动不要求管理员权限，启动后保持现有的右侧极细触发条状态，不自动展开完整面板。
- 每次程序启动时根据已保存设置校正启动项路径，使安装路径升级或移动后能够更新。
- 注册表写入失败时保持程序可用，撤销本次设置变化，并在界面状态提示中说明失败。
- 卸载器删除 `Horizon` 启动项，但保留用户任务数据。

## 结构

- `TrayIconService` 负责托盘图标、左键事件、右键菜单与资源释放，不承载窗口布局逻辑。
- `WindowsStartupService` 负责生成启动命令及读写当前用户启动项，不依赖界面。
- `MainWindow` 将托盘“打开”操作映射到现有完整面板切换，并在窗口关闭时释放托盘服务。
- `MainViewModel` 保存开机启动偏好，并通过启动服务同步系统状态。
- 项目启用 Windows Forms 以使用稳定的 `NotifyIcon`，其余界面仍保持 WPF。

## 兼容与错误处理

- 仅支持现有的 Windows 10/11 x64 安装目标。
- 托盘图标优先提取当前程序图标；提取失败时使用系统应用图标。
- 启动项不存在时关闭操作视为成功。
- 启动服务不得因为注册表异常导致 Horizon 无法启动。
- 不创建第二个隐藏窗口，不改变任务数据存储路径。

## 验收标准

- Horizon 运行时任务栏不出现应用按钮，通知区域存在图标。
- 左键单击托盘图标能够展开并激活完整面板。
- 右键菜单能够打开或真正退出 Horizon。
- 新安装首次启动后，“开机自动启动”默认开启且启动项存在。
- 关闭设置后启动项立即删除，重启 Horizon 后状态保持关闭。
- 再次开启后启动项指向当前安装目录下的 `Horizon.App.exe`。
- 开机启动时主窗口保持极细触发条，不弹出完整面板。
- 卸载后启动项被清理，`%LocalAppData%\Horizon\data` 中的任务数据保留。
- 自动化测试、Release 构建、自包含发布和安装器生成全部通过。

## 范围外

- 不增加托盘任务列表、提醒角标或通知推送。
- 不增加“关闭到托盘”按钮或任务栏最小化行为。
- 不实现多实例控制或跨设备同步。
