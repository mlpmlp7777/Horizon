# Horizon 任务批注设计

## 目标

每个本周任务和长期任务都可维护多条批注；主页面紧凑展示最新批注内容、时间和总数，完整批注列表支持追加、修改和删除。

## 数据模型

新增 `TaskAnnotation`：

- `Id`：稳定唯一标识。
- `Content`：去除首尾空白后的批注内容，长度为 `1–500` 个字符。
- `CreatedAt`：UTC 创建时间。
- `UpdatedAt`：UTC 最后修改时间。

`WeeklyTaskItem` 和 `LongTermTaskItem` 分别增加 `List<TaskAnnotation> Annotations`，默认空列表。旧 JSON 缺少该字段时按空列表读取，不需要复制或迁移任务。

## 主页面展示

- 任务卡片底部增加一条浅色分隔线。
- 分隔线下只显示按 `CreatedAt` 最新的一条批注，内容限制为单行并使用省略号。
- 同行右侧显示本地时间：当天使用“今天 HH:mm”，前一天使用“昨天 HH:mm”，更早使用 `yyyy-MM-dd HH:mm`。
- 下一行右对齐显示“共 N 条批注 ›”。
- 没有批注时不增加预览区域，只显示轻量的“添加批注”入口。
- 批注区域整体额外高度控制在 `34–38 px`，不改变标题、状态和进度的主视觉层级。

## 批注详情

- 点击批注预览或“添加批注”后，在现有应用内编辑层打开批注详情，不创建新窗口。
- 详情顶部显示所属任务标题和返回按钮。
- 批注按 `CreatedAt` 倒序排列。
- 支持新增、编辑和删除；新增和编辑使用同一个多行输入框。
- 编辑不改变 `CreatedAt`，只更新 `UpdatedAt`；已编辑批注显示“已编辑”。
- 删除使用卡片内二次确认：“确认删除 / 取消”，不使用系统弹窗。
- 保存、编辑和删除后立即写入本地 JSON，并刷新主页面预览和历史详情。
- 历史中的任务同样可以进入批注详情并进行增删改。

## 校验与异常

- 空内容或只有空白的内容不能保存。
- 超过 `500` 字符时显示明确校验提示，不截断用户输入。
- 目标任务不存在时关闭批注详情并显示状态提示，不写入数据。
- 删除或编辑时根据批注 ID 定位，避免列表刷新后作用到错误条目。

## 验证

- 本周任务和长期任务都能新增、编辑和删除批注。
- 主页面只显示最新批注、正确时间和总条数。
- 长内容使用单行省略号，任务卡片保持紧凑。
- 编辑批注后保留创建时间并显示“已编辑”。
- 删除必须经过二次确认。
- 重启后批注保持，旧数据可以正常打开。
- 主页面与历史详情显示同一份批注数据。
- 自动测试、Debug 构建和 Release 发布通过。

## 非目标

- 不增加批注作者、附件、提醒、富文本或云同步。
- 不支持批注排序自定义。
